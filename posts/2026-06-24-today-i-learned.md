# 오늘 배운 것

클로드 코드와 함께 블로그를 직접 만들어보면서 HTML, CSS, JavaScript가 각각 어떤 역할을 하는지 자연스럽게 배울 수 있었다.

## HTML — 뼈대를 세우다

HTML은 웹페이지의 **구조**를 담당한다. 제목, 본문, 링크, 버튼 같은 요소들을 배치하는 역할이다.

블로그를 만들면서 `<header>`, `<main>`, `<article>`, `<footer>` 같은 시맨틱 태그를 사용했는데, 이렇게 의미를 가진 태그를 쓰면 코드를 읽기도 쉽고 검색엔진이나 스크린 리더도 페이지를 더 잘 이해할 수 있다.

```html
<article id="content">
  <!-- 마크다운이 여기에 렌더링된다 -->
</article>
```

## CSS — 옷을 입히다

CSS는 **디자인**을 담당한다. 색상, 폰트, 여백, 레이아웃 등 눈에 보이는 모든 것을 제어한다.

특히 인상 깊었던 건 **CSS 변수**다. 색상 값을 변수로 정의해두면 다크 모드 전환을 깔끔하게 구현할 수 있다.

```css
:root {
  --bg: #ffffff;
  --text: #2c3e50;
}

[data-theme="dark"] {
  --bg: #1a1a2e;
  --text: #e0e0e0;
}
```

변수 값만 바꾸면 페이지 전체의 색상이 한꺼번에 바뀌니 정말 효율적이다.

## JavaScript — 생명을 불어넣다

JavaScript는 **동작**을 담당한다. 사용자의 클릭에 반응하고, 데이터를 불러오고, 페이지를 동적으로 변경하는 역할이다.

이 블로그에서는 세 가지 핵심 기능을 JavaScript로 구현했다:

- **마크다운 로딩** — `fetch`로 `.md` 파일을 불러와 화면에 표시
- **다크 모드 토글** — 버튼 클릭 시 테마를 전환하고 `localStorage`에 저장
- **글 목록 렌더링** — JSON 데이터를 읽어 글 목록을 동적으로 생성

## 정리

| 기술 | 역할 | 비유 |
|------|------|------|
| HTML | 구조 | 건물의 골조 |
| CSS | 디자인 | 인테리어와 외관 |
| JavaScript | 동작 | 전기와 배관 |

세 가지가 각자의 역할을 맡아 하나의 웹사이트를 완성한다. 프레임워크 없이 직접 만들어보니 각 기술이 왜 필요한지 훨씬 선명하게 이해할 수 있었다.
