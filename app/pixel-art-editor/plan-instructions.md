# Plan 서브에이전트 지침: 픽셀 아트 에디터

## 역할
너는 이 블로그 프로젝트의 "Plan" 단계를 담당하는 서브에이전트다. 아래 요구사항을 바탕으로
`app/pixel-art-editor/spec.md` 기획서를 작성하는 것이 유일한 임무다. **코드를 작성하지 마라.**
구현은 이후 별도의 Build 서브에이전트가 담당한다.

## 앱 요구사항 (사용자 요청)
- 16x16 격자에 도트를 찍어서 그림을 그리는 픽셀 아트 에디터
- 완성한 그림을 PNG 파일로 저장(다운로드)할 수 있어야 함
- 색상 팔레트 UI 필요 (여러 색상 중 선택해서 그리기)

## 참고 자료
- 기존 앱 예시(형식 참고용): `app/2048/spec.md` — 이 문서와 동일한 구조/수준의 상세도로 작성할 것
  (앱 개요, 기능 목록, 파일 구조, 기술 스택, UI 설계, 모바일 지원 섹션 포함)
- 블로그 전체 규칙: 프로젝트 루트 `CLAUDE.md` 참고
  - 모든 웹앱은 `/app/{앱이름}/` 폴더 안에 자체 완결
  - 외부 라이브러리 최소화 (CDN 허용)
  - 모바일에서도 사용 가능해야 함
  - 블로그의 다른 파일(index.html, css/, js/ 등)은 건드리지 말 것

## spec.md에 반드시 포함할 내용

1. **앱 개요**
2. **기능 목록**
   - 그리드/캔버스: 16x16 격자, 각 셀 클릭/드래그로 색칠
   - 드로잉 도구: 펜(칠하기), 지우개, (선택) 칠 채우기(bucket fill)
   - 색상 팔레트: 프리셋 색상 목록 + 커스텀 색상 선택(color picker `<input type="color">`)
   - 현재 선택된 색상 표시
   - 전체 지우기(Clear) 기능
   - PNG로 저장(다운로드) 기능 — `<canvas>`의 `toDataURL` 또는 `toBlob` 활용, 16x16 원본 해상도를 실제 저장 시 적절한 배율로 확대 저장할지 여부 결정 (예: 16배 확대하여 256x256으로 저장 등 픽셀이 뭉개지지 않게)
   - (선택) undo 기능은 범위에 따라 포함 여부 결정, 필수는 아님
3. **파일 구조**: `app/pixel-art-editor/` 하위에 index.html, style.css, editor.js(또는 app.js) 등
4. **기술 스택**: Vanilla HTML/CSS/JS, `<canvas>` API 사용 여부 결정 (grid를 canvas로 그릴지 CSS Grid + div로 그릴지 명확히 정하고 근거 서술). PNG 저장을 위해서는 결국 canvas가 필요하므로, 편집도 canvas 기반으로 할지 DOM 기반 후 저장시에만 canvas로 변환할지 설계 결정을 명확히 할 것.
5. **UI 설계**: 레이아웃 스케치(ASCII), 색상 팔레트 배치, 도구 버튼 배치, 다크모드 대응(블로그 본체와 동일한 방식: `prefers-color-scheme` + `[data-theme]`)
6. **모바일 지원**: 터치로 드롯 찍기/드래그 드로잉 지원, 반응형 크기 조절, 최소 터치 영역
