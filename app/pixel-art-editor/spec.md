# 픽셀 아트 에디터 기획서

## 1. 앱 개요

16x16 격자에 도트를 찍어 그림을 그리고, 완성한 그림을 PNG 파일로 저장(다운로드)할 수 있는 미니 픽셀 아트 에디터. 펜/지우개/채우기 도구와 색상 팔레트를 제공하며, 편집과 저장 모두 `<canvas>` 하나로 처리한다.

---

## 2. 기능 목록

### 2.1 그리드/캔버스

- 16x16 논리 픽셀 격자를 `<canvas>` 하나로 표현
- 캔버스는 CSS 상 큰 크기(예: 최대 512x512px, 반응형으로 축소)로 확대 렌더링하되, `imageSmoothingEnabled = false`로 픽셀 경계를 또렷하게 유지
- 각 셀은 클릭 시 즉시 색칠, 마우스/터치 드래그 시 지나가는 셀이 연속으로 칠해짐 (같은 셀 중복 클릭으로 인한 불필요한 재드로잉은 방지)
- 격자선을 얇게 표시해 셀 구분을 돕는다 (옵션: 격자선 표시/숨김은 필수 아님, 기본은 표시)

### 2.2 드로잉 도구

- **펜(칠하기)**: 현재 선택된 색상으로 셀을 칠함
- **지우개**: 셀을 투명/배경색으로 되돌림
- **칠 채우기(bucket fill)**: 클릭한 셀과 연결된 동일 색상 영역을 현재 색상으로 일괄 변경 (flood fill, 4방향 인접 기준)
- 현재 선택된 도구를 시각적으로 표시 (활성 버튼 강조)

### 2.3 색상 팔레트

- 프리셋 색상 목록: 대표 색상 12~16개 정도를 버튼 형태로 배치 (검정, 흰색, 회색, 빨강, 주황, 노랑, 초록, 청록, 파랑, 남색, 보라, 분홍, 갈색 등)
- 커스텀 색상 선택: `<input type="color">` 로 임의의 색상 선택 가능
- 현재 선택된 색상을 별도 스와치(작은 사각형)로 화면에 표시

### 2.4 편집 제어

- **전체 지우기(Clear)**: 확인 후 캔버스 전체를 초기 상태(투명/배경색)로 리셋
- **PNG로 저장**: 16x16 원본 해상도를 16배 확대한 256x256 PNG로 다운로드. 화면 표시용 캔버스와 별도로, 저장 시점에 오프스크린 캔버스(256x256, `imageSmoothingEnabled = false`)에 16x16 픽셀 데이터를 다시 그린 뒤 `toDataURL('image/png')` 또는 `toBlob`으로 다운로드 링크 생성
- **undo(선택)**: 범위에 따라 포함 여부 결정. 필수 기능은 아니며, 시간이 허락하면 간단한 1단계 undo(직전 상태 스택) 정도만 고려. 없어도 무방.

## 3. 파일 구조

```
app/pixel-art-editor/
├── index.html      # 에디터 페이지 (자체 완결 HTML)
├── style.css       # 에디터 전용 스타일
├── editor.js        # 캔버스 렌더링, 드로잉 로직, 팔레트/도구 이벤트, PNG 저장
└── spec.md          # 이 기획서
```

### 파일별 역할

- **index.html**: 에디터 레이아웃 마크업 (캔버스, 도구 버튼, 팔레트, 저장/초기화 버튼). style.css, editor.js를 로드. 블로그 본체와 독립적으로 단독 실행 가능.
- **style.css**: 캔버스 래퍼, 도구바, 팔레트, 색상 스와치, 버튼 등 모든 스타일. CSS 변수로 색상 관리. 다크 모드 대응.
- **editor.js**: 단일 스크립트로 상태(16x16 픽셀 데이터 배열, 현재 색상, 현재 도구)와 렌더링, 이벤트 처리, PNG 저장을 모두 담당. 2048과 달리 로직/UI를 분리할 만큼 복잡하지 않으므로 파일 하나로 충분.

## 4. 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 마크업 | HTML5 | 시맨틱 태그 사용 |
| 스타일 | CSS3 | CSS 변수, Flexbox/Grid, 다크모드 대응 |
| 로직 | Vanilla JavaScript (ES6+) | 외부 라이브러리 없음 |
| 렌더링 | `<canvas>` 단일 캔버스 | DOM 격자(div x 256개) 대신 canvas 하나로 편집·저장을 모두 처리 |
| 외부 의존성 | 없음 | CDN 포함 제로 의존성 |

### 설계 결정: canvas 기반 편집 (DOM grid 미사용)

PNG로 저장하려면 결국 `<canvas>`가 필요하다. 만약 편집을 DOM(div 256개의 CSS Grid)으로 하고 저장 시점에만 canvas로 변환한다면, 픽셀 데이터를 DOM과 canvas 양쪽에서 이중으로 관리해야 하고 동기화 버그 위험이 생긴다.

따라서 이 앱은 **처음부터 canvas 하나로 편집과 저장을 모두 처리**한다.

- **내부 상태**: 16x16 크기의 1차원 또는 2차원 배열에 각 셀의 색상(hex 문자열 또는 null=투명)을 저장. 이 배열이 단일 진실 소스(Single Source of Truth).
- **화면 표시**: 화면에는 큰 캔버스(예: `width: min(90vw, 512px)` 반응형, 내부 해상도는 CSS 크기에 맞춰 배율 설정)를 두고, 상태 배열을 순회하며 각 셀을 `fillRect`로 확대해서 그린다. `ctx.imageSmoothingEnabled = false`로 설정해 확대 시에도 픽셀이 흐려지지 않도록 한다.
- **입력 처리**: 마우스/터치 좌표를 캔버스의 실제 렌더 크기와 16x16 격자 비율로 환산해 어느 셀(행, 열)을 가리키는지 계산한 뒤, 상태 배열을 갱신하고 해당 부분만(또는 전체를) 다시 그린다.
- **PNG 저장**: 저장 버튼 클릭 시, 화면용 캔버스와는 별도로 오프스크린 캔버스(256x256 = 16배 확대)를 새로 만들어 상태 배열을 16배 픽셀 블록으로 다시 그리고, `imageSmoothingEnabled = false` 설정 후 `toDataURL('image/png')`로 다운로드 링크(`<a download>`)를 생성해 클릭한다. 화면 표시 캔버스의 CSS 크기와 저장용 배율은 독립적이므로, 화면 크기가 반응형으로 바뀌어도 저장 결과는 항상 256x256으로 고정된다.

## 5. UI 설계

### 5.1 레이아웃

```
┌───────────────────────────────────┐
│         픽셀 아트 에디터            │  <- 제목
│                                    │
│  [펜] [지우개] [채우기]             │  <- 도구 버튼
│                                    │
│  ┌────────────────────────────┐   │
│  │                            │   │
│  │                            │   │
│  │      16 x 16 캔버스         │   │  <- 편집 캔버스
│  │                            │   │
│  │                            │   │
│  └────────────────────────────┘   │
│                                    │
│  현재 색상: [■]                    │  <- 현재 색상 스와치
│                                    │
│  ■ ■ ■ ■ ■ ■ ■ ■                  │  <- 프리셋 팔레트
│  ■ ■ ■ ■ ■ ■ ■ ■                  │
│  [사용자 지정 색상: 🎨]              │  <- <input type="color">
│                                    │
│  [전체 지우기]   [PNG로 저장]        │  <- 편집 제어 버튼
└───────────────────────────────────┘
```

- 전체 레이아웃은 세로 중앙 정렬 (`max-width: 560px`, `margin: 0 auto`)
- 캔버스는 정사각형 비율 유지, 반응형 크기 (`width: min(90vw, 512px)`, `height`도 동일값으로 맞춤)
- 도구 버튼과 팔레트는 캔버스 위/아래에 배치, 좁은 화면에서는 자동 줄바꿈(`flex-wrap: wrap`)

### 5.2 색상 체계 (다크모드 대응)

블로그 본체(`css/style.css`)는 `:root`에 `--bg`, `--bg-secondary`, `--text`, `--text-secondary`, `--accent`, `--accent-hover`, `--border`, `--code-bg` 등을 정의하고, `[data-theme="dark"]` 클래스 선택자와 `@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) { ... } }` 두 군데에 동일한 다크 값을 중복 정의하는 방식을 쓴다. 다크모드 전환은 `js/main.js`의 `toggleTheme()`가 `document.documentElement`에 `data-theme="dark"|"light"`를 설정하고 `localStorage`에 저장하는 방식이다.

이 앱은 블로그 본체 `js/main.js`를 로드하지 않고 독립 실행되므로(2048과 동일한 방식), **에디터 자체 CSS 변수를 별도로 정의**하되 동일한 선택자 패턴(`[data-theme="dark"]` + `@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) }`)을 따른다. 앱 자체에도 다크모드 토글 버튼을 두어 `data-theme` 속성을 직접 제어한다 (2048의 방식과 동일).

#### 라이트 모드

| 변수 | 값(예시) | 용도 |
|------|-----|------|
| `--bg` | `#ffffff` | 페이지 배경 |
| `--bg-secondary` | `#f8f9fa` | 도구바/패널 배경 |
| `--text` | `#2c3e50` | 기본 텍스트 |
| `--text-secondary` | `#6c757d` | 보조 텍스트/안내문 |
| `--accent` | `#3498db` | 활성 도구 강조, 저장 버튼 |
| `--accent-hover` | `#2980b9` | 강조 요소 hover |
| `--border` | `#e9ecef` | 캔버스 테두리, 구분선 |
| `--canvas-grid-line` | `#dcdcdc` | 캔버스 내부 격자선 |
| `--canvas-bg` | `#ffffff` | 캔버스 빈 셀(투명 픽셀) 배경 |

#### 다크 모드 (`[data-theme="dark"]` 및 `prefers-color-scheme: dark`)

| 변수 | 값(예시) | 용도 |
|------|-----|------|
| `--bg` | `#1a1a2e` | 페이지 배경 (블로그 다크 모드와 통일) |
| `--bg-secondary` | `#16213e` | 도구바/패널 배경 |
| `--text` | `#e0e0e0` | 기본 텍스트 |
| `--text-secondary` | `#a0a0a0` | 보조 텍스트/안내문 |
| `--accent` | `#64b5f6` | 활성 도구 강조, 저장 버튼 |
| `--accent-hover` | `#90caf9` | 강조 요소 hover |
| `--border` | `#2a2a4a` | 캔버스 테두리, 구분선 |
| `--canvas-grid-line` | `#3a3a5a` | 캔버스 내부 격자선 |
| `--canvas-bg` | `#242440` | 캔버스 빈 셀(투명 픽셀) 배경 |

값은 블로그 본체 `css/style.css`의 `--bg`, `--bg-secondary`, `--text`, `--text-secondary`, `--accent`, `--accent-hover`, `--border` 값을 그대로 재사용해 시각적 통일감을 준다. 프리셋 팔레트 색상 자체(빨강/파랑 등)는 다크모드 여부와 무관하게 고정.

### 5.3 타이포그래피

- 제목: `bold 1.8rem`
- 버튼/라벨: `0.95rem`, 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`, 블로그 본체와 통일)

### 5.4 상호작용 디테일

- 도구 버튼(펜/지우개/채우기)은 토글형: 클릭 시 선택 상태 유지, 활성 버튼은 `--accent` 배경으로 강조
- 팔레트 색상 버튼 클릭 시 즉시 현재 색상 갱신 및 스와치 반영
- 커스텀 컬러피커 값 변경(`input` 이벤트) 시에도 동일하게 현재 색상 갱신
- 전체 지우기는 실수 방지를 위해 `confirm()` 또는 간단한 확인 UI를 거친 뒤 실행

## 6. 모바일 지원

### 6.1 반응형 레이아웃

- 캔버스 크기: `width: min(90vw, 512px)`의 정사각형, 뷰포트에 맞게 축소
- 도구바/팔레트도 `flex-wrap: wrap`으로 좁은 화면에서 자동 줄바꿈
- `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수

### 6.2 터치 드로잉

- `touchstart`, `touchmove`, `touchend` 이벤트로 드래그 드로잉 지원 (마우스의 `mousedown`/`mousemove`/`mouseup`과 동일한 셀 계산 로직 재사용)
- 터치 좌표는 `event.touches[0].clientX/clientY`를 캔버스의 `getBoundingClientRect()` 기준으로 환산
- 캔버스 위에서 드로잉 중 페이지 스크롤/핀치줌이 발생하지 않도록 `touch-action: none`을 캔버스에 적용하고 필요 시 `preventDefault()` 호출

### 6.3 터치 최적화

- 모든 버튼(도구, 팔레트 색상, 저장/초기화) 최소 터치 영역: 44px x 44px 이상
- 더블 탭 확대 방지: 주요 인터랙션 영역에 `touch-action: manipulation` 또는 `touch-action: none`(캔버스) 적용
- 팔레트 색상 버튼은 모바일에서도 손가락으로 오탭 없이 구분 가능한 크기(최소 32~40px, 간격 확보)로 배치
