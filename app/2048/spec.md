# 2048 퍼즐 게임 기획서

## 1. 앱 개요

4x4 격자에서 타일을 밀어 같은 숫자끼리 합치며 2048을 만드는 싱글 플레이어 퍼즐 게임.

---

## 2. 기능 목록

### 2.1 게임 로직

- **격자 초기화**: 4x4 빈 격자에서 시작하며, 랜덤한 위치에 2개의 타일(2 또는 4) 배치
- **타일 이동**: 상/하/좌/우 방향으로 모든 타일이 해당 방향 끝까지 이동
- **타일 병합**: 같은 숫자의 타일이 충돌하면 합쳐져 두 배 값의 타일이 됨 (2+2=4, 4+4=8, ...)
- **병합 규칙**: 한 번의 이동에서 하나의 타일은 최대 한 번만 병합 가능 (예: 2-2-2-2 -> 4-4가 되지, 8이 되지 않음)
- **새 타일 생성**: 이동 후 빈 칸 중 랜덤한 위치에 새 타일 생성 (90% 확률로 2, 10% 확률로 4)
- **승리 조건**: 2048 타일을 만들면 승리 메시지 표시, 이후 계속 플레이 가능
- **패배 조건**: 빈 칸이 없고 인접한 같은 숫자 타일도 없으면 게임 오버

### 2.2 점수 시스템

- **점수 계산**: 타일이 병합될 때마다 합쳐진 결과 값이 점수에 더해짐 (예: 4+4=8 병합 시 +8점)
- **현재 점수**: 진행 중인 게임의 누적 점수
- **최고 점수**: 브라우저 localStorage에 저장하여 세션 간 유지

### 2.3 UI 기능

- **새 게임 버튼**: 게임을 초기 상태로 리셋
- **점수판**: 현재 점수와 최고 점수를 나란히 표시
- **게임 오버 오버레이**: 반투명 오버레이로 "게임 오버" 메시지와 다시 시작 버튼 표시
- **승리 오버레이**: 2048 달성 시 축하 메시지와 "계속하기" / "새 게임" 버튼 표시
- **타일 애니메이션**: 이동과 병합 시 부드러운 CSS 트랜지션/애니메이션

### 2.4 입력 처리

- **키보드**: 방향키(ArrowUp/Down/Left/Right) 및 WASD 지원
- **터치 스와이프**: 모바일 기기에서 스와이프 제스처로 조작 (최소 이동 거리 임계값 적용)
- **입력 방지**: 애니메이션 진행 중 또는 게임 오버 상태에서 추가 입력 무시

## 3. 파일 구조

```
app/2048/
├── index.html      # 게임 페이지 (자체 완결 HTML)
├── style.css       # 게임 전용 스타일
├── game.js         # 게임 로직 (격자 상태, 이동, 병합, 승리/패배 판정)
├── ui.js           # DOM 렌더링, 애니메이션, 이벤트 바인딩
└── spec.md         # 이 기획서
```

### 파일별 역할

- **index.html**: 게임 레이아웃 마크업. style.css, game.js, ui.js를 로드. 블로그 본체와 독립적으로 단독 실행 가능.
- **style.css**: 격자, 타일, 점수판, 오버레이, 애니메이션 등 모든 스타일. CSS 변수로 색상 관리. 다크 모드 대응.
- **game.js**: 순수 로직 모듈. 4x4 배열 상태 관리, 이동/병합 알고리즘, 점수 계산, 승리/패배 판정. DOM에 의존하지 않음.
- **ui.js**: game.js의 상태를 DOM에 반영. 키보드/터치 이벤트 처리, 애니메이션 트리거, localStorage를 통한 최고 점수 관리.

## 4. 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 마크업 | HTML5 | 시맨틱 태그 사용 |
| 스타일 | CSS3 | CSS Grid, CSS 변수, 트랜지션/애니메이션 |
| 로직 | Vanilla JavaScript (ES6+) | 외부 라이브러리 없음 |
| 저장소 | localStorage | 최고 점수 저장 |
| 외부 의존성 | 없음 | CDN 포함 제로 의존성 |

## 5. UI 설계

### 5.1 레이아웃

```
┌─────────────────────────────┐
│         2048                │  <- 게임 제목
│                             │
│  ┌──────────┐ ┌──────────┐  │
│  │ SCORE    │ │ BEST     │  │  <- 점수판 (가로 나란히)
│  │   1234   │ │   5678   │  │
│  └──────────┘ └──────────┘  │
│                             │
│  [새 게임]                   │  <- 새 게임 버튼
│                             │
│  ┌──┬──┬──┬──┐              │
│  │  │  │  │  │              │
│  ├──┼──┼──┼──┤              │
│  │  │  │  │  │              │  <- 4x4 게임 격자
│  ├──┼──┼──┼──┤              │
│  │  │  │  │  │              │
│  ├──┼──┼──┼──┤              │
│  │  │  │  │  │              │
│  └──┴──┴──┴──┘              │
│                             │
│  방향키 또는 스와이프로 조작  │  <- 안내 텍스트
└─────────────────────────────┘
```

- 전체 레이아웃은 세로 중앙 정렬 (`max-width: 500px`, `margin: 0 auto`)
- 격자는 정사각형 비율 유지 (`aspect-ratio: 1` 또는 `padding-bottom` 트릭)
- 격자 내부는 CSS Grid (`grid-template-columns: repeat(4, 1fr)`, `gap`으로 간격)

### 5.2 색상 체계

CSS 변수를 사용하여 라이트/다크 모드 모두 지원한다.

#### 라이트 모드

| 변수 | 값 | 용도 |
|------|-----|------|
| `--game-bg` | `#faf8ef` | 페이지 배경 |
| `--grid-bg` | `#bbada0` | 격자 배경 |
| `--cell-bg` | `#cdc1b4` | 빈 칸 배경 |
| `--text-dark` | `#776e65` | 어두운 타일(2, 4)의 텍스트 |
| `--text-light` | `#f9f6f2` | 밝은 타일(8 이상)의 텍스트 |

#### 타일 색상 (라이트 모드 기준, 다크 모드에서도 동일하게 유지)

| 타일 값 | 배경색 | 텍스트색 |
|---------|--------|----------|
| 2 | `#eee4da` | `--text-dark` |
| 4 | `#ede0c8` | `--text-dark` |
| 8 | `#f2b179` | `--text-light` |
| 16 | `#f59563` | `--text-light` |
| 32 | `#f67c5f` | `--text-light` |
| 64 | `#f65e3b` | `--text-light` |
| 128 | `#edcf72` | `--text-light` |
| 256 | `#edcc61` | `--text-light` |
| 512 | `#edc850` | `--text-light` |
| 1024 | `#edc53f` | `--text-light` |
| 2048 | `#edc22e` | `--text-light` |

#### 다크 모드

| 변수 | 값 | 용도 |
|------|-----|------|
| `--game-bg` | `#1a1a2e` | 페이지 배경 (블로그 다크 모드와 통일) |
| `--grid-bg` | `#3d3a33` | 격자 배경 |
| `--cell-bg` | `#4a4740` | 빈 칸 배경 |

다크 모드 감지: `prefers-color-scheme: dark` 미디어 쿼리 + `[data-theme]` 속성 수동 토글 지원 (블로그 본체와 동일한 방식).

### 5.3 타이포그래피

- 게임 제목: `bold 2.5rem`
- 타일 숫자: `bold`, 자릿수에 따라 크기 조절
  - 1자리 (2~8): `2rem`
  - 2자리 (16~64): `1.8rem`
  - 3자리 (128~512): `1.5rem`
  - 4자리 (1024~): `1.2rem`
- 폰트: 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)

### 5.4 애니메이션

- **타일 등장**: `scale(0) -> scale(1)`, 150ms ease-in-out
- **타일 이동**: `transform: translate()`, 100ms ease-in-out
- **타일 병합**: 이동 완료 후 `scale(1) -> scale(1.1) -> scale(1)`, 100ms 펄스 효과
- **오버레이 등장**: `opacity: 0 -> 1`, 300ms ease

## 6. 모바일 지원

### 6.1 반응형 레이아웃

- 격자 크기: 뷰포트 너비 기준 `min(90vw, 500px)` 정사각형
- 점수판, 버튼도 뷰포트에 맞게 축소
- 불필요한 스크롤 방지: 전체 UI가 화면 안에 들어오도록 설계
- `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수

### 6.2 터치 스와이프

- `touchstart`, `touchend` 이벤트로 스와이프 방향 감지
- 시작 좌표와 종료 좌표의 차이로 방향 결정 (가로/세로 중 이동 거리가 큰 쪽)
- 최소 임계값: 30px 이상 이동해야 스와이프로 인식
- 게임 중 기본 스크롤 동작 방지 (`preventDefault`)

### 6.3 터치 최적화

- 버튼 최소 터치 영역: 44px x 44px
- 더블 탭 확대 방지: `touch-action: manipulation`
- 게임 격자 영역에서 브라우저 기본 제스처(당겨서 새로고침 등) 방지
