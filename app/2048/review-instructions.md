# 2048 게임 Review 지침

## 목표
Build 서브에이전트가 구현한 2048 게임이 spec.md 사양대로 정상 작동하는지 검증한다.

## 검증 범위
`app/2048/` 폴더의 다음 파일:
- index.html, style.css, game.js, ui.js

## 검증 항목

### 1. 코드 품질
- game.js가 DOM에 의존하지 않는 순수 로직인지 확인
- ui.js와 game.js의 역할이 올바르게 분리되어 있는지 확인
- 외부 라이브러리 없이 순수 JS로 구현되었는지 확인

### 2. 게임 로직
- 타일 이동/병합이 정확한지 (한 번의 이동에서 타일은 최대 한 번만 병합)
- 새 타일 생성 (90% 확률 2, 10% 확률 4)
- 승리 조건 (2048 타일) 및 패배 조건 (이동 불가) 판정
- 점수 계산 (병합 결과값 누적)

### 3. UI/UX
- 브라우저에서 preview 도구를 사용하여 실제 동작 확인
- 점수판이 올바르게 표시되는지
- 오버레이(승리/패배)가 정상 작동하는지
- 새 게임 버튼이 작동하는지

### 4. 반응형/접근성
- 모바일 뷰포트에서 레이아웃이 깨지지 않는지
- 다크 모드가 적용되는지

## 산출물
검증 결과를 `app/2048/review.md`에 작성한다.
- 각 항목별 통과/실패 여부
- 발견된 문제가 있으면 상세 설명
- 문제가 있으면 직접 수정한다

## 규칙
- `app/2048/` 폴더 안의 파일만 수정 가능
- 블로그 본체 파일은 절대 건드리지 않는다
